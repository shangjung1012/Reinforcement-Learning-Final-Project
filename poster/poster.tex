\begin{column}{\colwidth}

  \begin{block}{Abstract}
    This project treats RAG retrieval as a \textbf{cost-aware offline contextual bandit}: for each query, a lightweight policy chooses BM25, dense, hybrid, or rewrite actions from query and retrieval-confidence features. The main result is improved full-corpus evidence retrieval on \textbf{SciFact} and \textbf{NFCorpus}; generated-answer EM/F1 is reported only as a small Gemini-reader pilot.
  \end{block}

  \begin{alertblock}{Problem and Objective}
    Fixed RAG retrieval cannot adapt to query difficulty. Rewrites, dense retrieval, hybrid retrieval, or LLM experts can help, but only when their expected retrieval gain justifies cost.

    \begin{itemize}
      \item Learn a per-query retrieval/rewrite router from offline rewards.
      \item Keep cost explicit through rewrite and retrieval-call penalties.
      \item Validate claims with baselines, bootstrap intervals, OPE, and artifact checks.
    \end{itemize}
  \end{alertblock}

  \begin{block}{Contextual-Bandit Formulation}
    Each query is a one-step decision:

    \[
      \pi(s) = \arg\max_a \mathbb{E}[R \mid s, a]
    \]

    \vspace{-0.5em}
    \[
      R = \mathrm{Recall@5} + 0.5 \cdot \mathrm{MRR}
          - \mathrm{RewriteCost} - \mathrm{ExtraRetrievalCallCost}
    \]

    \begin{itemize}
      \item \textbf{State:} query features + BM25 confidence statistics.
      \item \textbf{Actions:} BM25, dense, hybrid, keyword rewrite; generated-query actions are optional expensive extensions.
      \item \textbf{Policy:} direct-method action-value models with CV model selection.
      \item \textbf{Training evidence:} replayable full-information retrieval rewards; this is not online bandit learning, and selected-action replay is a diagnostic baseline.
    \end{itemize}
  \end{block}

\begin{column}{\colwidth}

  \begin{block}{Datasets and Evidence Boundary}
    Final claims use seed-42 full-corpus BEIR runs with 600 train queries and 300 test queries per dataset: \textbf{SciFact} scientific-claim retrieval over 5,183 documents and \textbf{NFCorpus} biomedical/nutrition retrieval over 3,633 documents. Smoke tests verify code paths only.
  \end{block}

  \begin{block}{Main Results}
    \input{tables/table1}

    \vspace{0.35em}
    \input{images/figure2}

    The learned policy improves reward over the train-selected fixed action on both BEIR datasets, closing 22.6\% of the SciFact oracle gap and 34.4\% of the NFCorpus oracle gap. The constrained policy keeps positive utility gains while making call tradeoffs explicit.
  \end{block}

  \begin{block}{Diagnostics That Support the Claim}
    \begin{itemize}
      \item \textbf{Selected-action replay:} Linear Thompson delta is +0.005 on SciFact and +0.027 on NFCorpus.
      \item \textbf{Replay regret:} LinUCB selected-feedback regret is 82.1 vs 67.4 full-information direct on SciFact, and 43.2 vs 29.6 on NFCorpus.
      \item \textbf{OPE coverage warning:} SciFact IPS error rises from 0.127 under uniform logging to 0.268 under sparse train-best logging.
      \item \textbf{Three-seed robustness:} selective win rate is 0.667 on both datasets; mean deltas are +0.021 on SciFact and +0.025 on NFCorpus.
      \item \textbf{Audit trail:} markdown consistency, evidence consistency, checkpoint manifest, and artifact index pass.
    \end{itemize}
  \end{block}

  \begin{block}{Limitations and Claim Boundary}
    \begin{itemize}
      \item Final claim is evidence retrieval only; Gemini reader EM/F1 is pilot-only.
      \item No LLM fine-tuning or production RAG deployment.
      \item Direct method uses replayable full-information action tables.
      \item Not online bandit learning; OPE is simulated, not real online logs; semantic features remain analysis-only.
    \end{itemize}
  \end{block}
